## 🎯 Mission

To provide an opinionated, project-based, and AI-enhanced learning path for experienced software engineers to become proficient in idiomatic Rust for backend development, **fast**.

## 🤔 Why This Repository Exists

Learning a new language, especially one as powerful and strict as Rust, can be challenging. Standard tutorials often focus too much on basic syntax, while advanced textbooks can be overly theoretical. This repository is different. It's built on a few core principles:

1.  **The 80/20 Rule:** We focus on the 20% of Rust's features that you will use 80% of the time when building backend systems (Ownership, `Result`, `async`, `struct`s, `enum`s, `trait`s).

2.  **Active Learning:** You don't learn Rust by watching videos; you learn it by fighting the compiler and seeing your code run. This is an interactive sandbox, not a passive lecture.

3.  **AI as a Mentor:** The analysis and documentation within this repository were generated by a large language model (Google's Gemini) acting as a world-class Rust instructor. This entire repository is a testament to a new way of learning: using AI as a personalized, on-demand expert.

## ⚡ Quick Start: Easy Navigation Scripts

This repository includes **easy-to-use navigation scripts** to help you quickly jump to any lesson. No more deep folder diving!

- **PowerShell Users (Windows):** Use `rust_nav.ps1`
- **Bash Users (Linux/Mac):** Use `rust_nav.sh`

### 🔧 Setup Instructions

#### For PowerShell (Windows)

**Step 1: Enable Script Execution (One-Time)**

Open PowerShell as Administrator and run:

```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

Confirm with `Y` when prompted.

**Step 2: Add Function to Your PowerShell Profile**

Edit your PowerShell profile:

```powershell
notepad $PROFILE
```

If the file doesn't exist, click "Create" when prompted. Add these lines to the end:

```powershell
# Rust Masterclass Navigation Function
function nav {
    & "C:\Users\$env:USERNAME\Documents\projects\rust-backend-foundry\rust_nav.ps1" @args
}
```

**Important:** Replace the path if your rust-backend-foundry folder is in a different location.

**Step 3: Reload Your Profile**

Either restart PowerShell or run:

```powershell
. $PROFILE
```

**Usage Examples:**

```powershell
nav list          # List all modules and lessons
nav goto 1.1      # Jump to Module 1, Lesson 1
nav goto 6.2      # Jump to Module 6, Lesson 2
nav help          # Show help
```

---

#### For Bash/Zsh (Linux/Mac)

**Step 1: Make the Script Executable**

```bash
chmod +x ~/Documents/projects/rust-backend-foundry/rust_nav.sh
```

**Step 2: Add Alias to Your Shell Profile**

Edit your shell profile:

```bash
# For Bash:
nano ~/.bashrc

# For Zsh:
nano ~/.zshrc
```

Add these lines to the end:

```bash
# Rust Masterclass Navigation
alias nav='source ~/Documents/projects/rust-backend-foundry/rust_nav.sh'
```

**Important:** Adjust the path if your rust-backend-foundry folder is in a different location.

**Step 3: Reload Your Profile**

Either restart your terminal or run:

```bash
source ~/.bashrc   # or ~/.zshrc
```

**Usage Examples:**

```bash
nav list          # List all modules and lessons
nav goto 1.1      # Jump to Module 1, Lesson 1
nav goto 6.2      # Jump to Module 6, Lesson 2
nav help          # Show help
```

## 🚀 How to Use This Curriculum

To get the most out of this repository, follow this process strictly. Do not just read the code.

1.  **Read the Learning Guide:** Start with [How to Master Rust with This Repo](how_to_master_rust_with_this_repo.md) to understand the pedagogical approach.

2.  **Set Up Navigation:** Follow the instructions above to enable the `nav` command in your terminal for quick lesson access.

3.  **Navigate Sequentially:** Use `nav goto 1.1` to jump to Module 1, Lesson 1, then proceed in numerical order. Each lesson builds on the last.

4.  **Run the Code:** Before reading anything else, run the code using `cargo run`. Observe the output. If it fails, try to understand the compiler error before moving on.

5.  **Read the Analysis:** After running the code, open and read the `analysis.md` file within that lesson's directory. This is the "instructor commentary"—a detailed breakdown of the concepts, cross-language comparisons, and practical reflections.

6.  **Engage and Reflect:** Each `analysis.md` contains "Self-Review Prompts." Take a moment to think about them. This active recall is crucial for cementing your understanding.

7.  **Review the Summaries:** After completing all modules, read the summary documents ([Curriculum Summary](curriculum_summary.md), [Rust Mastery Phase 2](rust_mastery_phase_2.md), etc.) to synthesize your knowledge.

## 🤖 The AI-Powered Instructor: Understanding `gemini.md`

This repository contains a unique file: [gemini.md](gemini.md). This is not just documentation; it is a **prompt-as-config** file. It contains the high-level instructions that were fed to the Gemini AI to generate all the `analysis.md` files, the curriculum summary, and the roadmaps.

**This is a meta-lesson.** It teaches you not only Rust, but also how to leverage modern AI tools to create your own bespoke learning experiences. By studying [gemini.md](gemini.md), you can learn how to prompt an AI to act as a domain expert, a code analyzer, and a curriculum creator.

### Adapting for Other AI Tools (Claude, Codey, etc.)

The power of the [gemini.md](gemini.md) file is that the instructions are largely model-agnostic. You can adapt this repository to be powered by any capable AI model or CLI tool.

**To do so:**

1.  **Identify the Core Prompt:** Open [gemini.md](gemini.md) and copy the core instructional text—the sections defining the objective, traversal logic, output instructions, and desired persona.

2.  **Adapt the Invocation:**
    *   If you are using another AI CLI tool that accepts a prompt file (e.g., `claude run --prompt-file ...`), you can simply point it to [gemini.md](gemini.md).
    *   If you are using a web-based AI chat interface, you can paste the core instructions from [gemini.md](gemini.md) into the chat window as a large context-setting prompt.

3.  **Regenerate:** Ask the new AI to begin the process, starting with Lesson 1. It will follow the same instructions to regenerate the analysis files, but with its own unique voice, insights, and style. You can effectively create your own fine-tuned version of this course with the AI instructor of your choice.

## 🎓 Learning Philosophy

This curriculum is built on active learning principles:

- **Embrace the Struggle:** View compiler errors as conversations with your mentor, not roadblocks.
- **Understand the "Why":** Focus on principles, not just syntax.
- **Type It Out:** Don't copy-paste—muscle memory matters.
- **Experiment Boldly:** Break code intentionally to learn boundaries.
- **Build Projects:** Apply concepts to real-world problems.

For a comprehensive guide, see [How to Master Rust with This Repo](how_to_master_rust_with_this_repo.md).
